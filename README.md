# UI-UX-Experiment6

## Aim 

To analyze and represent the user journey for completing a task (booking a doctor appointment) using a flow diagram

To understand the sequence of user actions, decisions, and system responses

To visually map the process using standard flowchart symbols (start, process, decision, end)

To identify possible user paths and improve usability of the system

## Algorithm

1.Start 

2.Open the application

3.Login or continue as guest

4.Select user profile (self/family)

5.Search doctor or enter symptoms

6.Display doctor list

7.Select doctor

8.Check availability

9.If slot available → select slot

10.Else → choose another doctor

11.Confirm booking

12.Make payment (if required)

13.Show booking confirmation

14.End
